SplitEasy — Privacy Policy

Last updated: 19 May 2026

About SplitEasy

SplitEasy is a personal-use Progressive Web App for splitting trip expenses, tracking currency rates, and optionally importing transactions from linked bank accounts. It is built and operated by an individual for their own informal use; it is not a commercial product and there is no company behind it. By using the app you agree to the data handling described here.

What we collect

Only data necessary for the app to work:

  • What you create: trip names, member names and optional emails, expense descriptions, amounts, currencies, payments, receipt photos, currency watchlist, and rate alerts.
  • Authentication: your email address, used to send you a magic link or one-time code for sign-in. No password is stored — sign-in is passwordless.
  • Linked bank data (optional, via Plaid): bank name, account names, balances, and transactions for the date ranges you request. Plaid handles all bank logins through each bank's own OAuth flow — your bank password is never seen, transmitted, or stored by SplitEasy or Plaid.
  • Receipt scans (optional, via Claude AI): when you use the AI receipt scan, the receipt photo is sent to Anthropic's Claude API to extract items, totals, and merchant info. Anthropic does not train on submitted images.

Where data is stored

  • On your device: in your browser's local storage. Cleared when you clear browser data.
  • In Supabase (only if you sign in): a single JSON row keyed to your user ID, protected by Row Level Security so only your authenticated session can read it.

Third parties

SplitEasy uses these services:

  • Supabase — database and authentication.
  • Plaid — bank linking, only when you use the Bank tab.
  • Anthropic — Claude AI receipt scanning, only when you scan a receipt.
  • Frankfurter and ExchangeRate-API — read-only public currency exchange rates. No personal data sent.

No advertising, no analytics, no profile building, no data sold or shared with anyone else.

How we keep data safe

  • All connections use HTTPS (TLS).
  • Supabase rows are protected by Row Level Security — only the row owner's authenticated session can read or write.
  • Plaid access tokens are stored in Supabase under the same Row Level Security policy. If you sign out, the cloud copy remains tied to your account; you can clear it at any time.
  • SplitEasy never sees or stores your bank login credentials.

Your rights

  • Export a full backup of your data as JSON from Profile → Export backup.
  • Delete all locally stored data at any time from Profile → Clear all data.
  • Unlink any bank from the Bank tab at any time. This removes the access token from your record. Plaid retains some metadata according to its own policy.
  • Sign out at any time from Profile.
  • Account deletion: contact the developer (below) to delete your Supabase account record entirely.

Children

SplitEasy is not directed at users under 13. We do not knowingly collect data from children.

Changes to this policy

If this policy changes, the "Last updated" date above will change. Continued use of the app after that date constitutes acceptance.
